# State Page QA And Launch Gates

Run this checklist separately for every state.

## Evidence And Mode

- [ ] Current service mode is supported by evidence and human approval.
- [ ] Conflicting older repo claims are recorded.
- [ ] Active state has a verified intake and follow-up path.
- [ ] Pending state does not enter quote, booking, Sarah, or active nurture.

## Content And Compliance

- [ ] No invented state laws, statistics, offices, testimonials, or carrier
  availability.
- [ ] No guaranteed approval, exact pricing, universal qualification, or
  unsupported return/tax claims.
- [ ] Brokerage identity and state availability are clear.
- [ ] Required disclosure is present.
- [ ] Privacy and Terms links work.
- [ ] CTA language matches the service mode.

## UX And Visual

- [ ] Desktop layout reviewed.
- [ ] Mobile layout reviewed.
- [ ] Focus states and keyboard navigation reviewed.
- [ ] Primary CTA is clear without artificial urgency.
- [ ] State personalization feels meaningful rather than token replacement.
- [ ] Page matches Evermore navy, gold, typography, and narrative system.

## Technical And SEO

- [ ] Generator completes without validation errors.
- [ ] Title, description, canonical, Open Graph URL, and state name are correct.
- [ ] Draft and pending pages remain `noindex, nofollow`.
- [ ] No broken local asset or internal links.
- [ ] Live route is verified after approval and publishing.
- [ ] Analytics and conversion events are verified after approval and
  publishing.

## Approval-Gated Launch

- [ ] Human approved publishing.
- [ ] Compliance reviewer approved copy.
- [ ] State-specific workflow routing tested end to end.
- [ ] Live page and final CTA destination verified.
- [ ] Blueprint report updated with live proof.
